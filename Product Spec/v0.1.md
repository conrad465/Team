# WhiteBoard AI — Product Spec v0.1

*Last updated: 2026-03-09. Owner: PM skill + human.*

---

## Current State (PoC)

The end-to-end flow today is entirely manual:
1. User copies the system prompt from `Codebase/public/system-prompt.md`
2. Pastes it plus a topic into an external chatbot (Claude, ChatGPT, etc.)
3. Copies the returned JSON
4. Pastes it into the app on GitHub Pages
5. Watches the presentation

This works as a proof of concept but is too slow and friction-heavy to share with classmates. Estimated time from idea to playing presentation: 5–10 minutes.

---

## v1 Target Experience

A user arrives at the app, types a topic, and watches a presentation. No copy-paste, no external tools.

**Target: < 90 seconds from topic to playing presentation.**

### User Journey
1. User opens the app
2. Types a topic (one sentence or less)
3. LLM generates the scene JSON in the background
4. Presentation plays automatically with TTS narration
5. User can replay

---

## Success Metrics

| Metric | Target |
|---|---|
| Time from topic input to playback | < 90 seconds |
| Presentation intelligibility | Viewer understands the concept without prior knowledge |
| Repeat use | User generates > 1 presentation per session |
| Share intent | User says they'd share it with a peer |

---

## Scope

### In (v1)
- Integrated topic input → LLM call → render pipeline (no manual JSON step)
- Business/economics topics as the primary tested use case
- Single-scene presentations

### Out (v1)
- Editing or customizing generated presentations
- Saved history or user accounts
- Sharing infrastructure (links, embeds)
- Multi-scene / chapter structure

---

## Open Questions

1. **Where does the LLM call happen?** Client-side (API key in browser) or server-side? This affects latency, cost, and API key exposure.
2. **Which model?** Determines JSON conformance reliability and generation speed.
3. **How reliable is JSON conformance?** Has the system prompt been tested across a range of business school topics? What's the failure rate?
4. **TTS source?** Which TTS engine is currently used, and does it sound natural enough for classmates?

---

## Risks

| Risk | Mitigation |
|---|---|
| LLM produces invalid JSON → broken render | Schema validation with a user-facing error message; test system prompt on 10+ diverse topics before launch |
| Generation latency > 90s | Measure end-to-end time now; streaming render if possible |
| Presentations feel generic or unimpressive | Test with real business school topics before sharing; iterate on system prompt |
| Classmates can't figure out how to use it | First-run experience must require zero explanation |

---

## Definition of "Ready for Classmates"

- [ ] Integrated input → generation → playback flow (no manual JSON step)
- [ ] Tested on 5+ business school topics with acceptable output quality
- [ ] Works on GitHub Pages without setup or instructions
- [ ] Generation completes in < 90 seconds on a normal connection
